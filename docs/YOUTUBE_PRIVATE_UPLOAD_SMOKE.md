# YouTube Private Upload Smoke

This runbook covers one approval-gated YouTube private or unlisted upload smoke from a local mp4 file.

It does not enable public upload, TikTok upload, Threads upload, DB writes, R2 uploads, queue creation, worker job creation, or upload package creation.

## Required Gates

All gates must be true before `POST /api/uploads/youtube/execute` is attempted:

- `RUN_YOUTUBE_PRIVATE_UPLOAD_SMOKE` is configured with the exact smoke phrase.
- Request body confirmation is exactly `APPROVE_YOUTUBE_PRIVATE_UPLOAD`.
- YouTube readiness returns `can_upload=true`.
- Token readiness returns `token_ready=true` and `scopes_ready=true`.
- Visibility is `private` or `unlisted`.
- `PUBLIC_UPLOAD_ENABLED` is not true.
- `video_path_or_url` points to an existing local `.mp4` file.
- `disclosure_text` is present.
- `selected_affiliate_url` is present.

## Token File Env Compatibility

The local token provider supports both names:

1. `YOUTUBE_LOCAL_TOKEN_FILE_PATH`
2. `YOUTUBE_TOKEN_FILE`

`YOUTUBE_LOCAL_TOKEN_FILE_PATH` takes priority when both are configured. The token file must be outside this repository.

Do not print token values, token JSON, client secrets, or Authorization headers.

## Local MP4 Preparation

Preferred path:

```text
commerce-assets/output/video-packages/youtube-private-smoke-001/youtube-private-smoke-001.mp4
```

The mp4 may be generated by the existing local slideshow render execution flow. Generated mp4 files must remain untracked and must not be committed.

## Prepare Request

Run `POST /api/uploads/youtube/prepare` first. Expected side effects:

```json
{
  "external_api_called": false,
  "youtube_upload_executed": false,
  "uploaded": false,
  "db_written": false,
  "r2_uploaded": false,
  "queue_created": false,
  "worker_job_created": false,
  "platform_upload_triggered": false,
  "public_upload_enabled": false
}
```

If prepare fails, do not execute upload.

## Execute Request

`POST /api/uploads/youtube/execute` starts a server-only YouTube Data API resumable upload only after every gate passes.

Success requires:

- `attempted=true`
- `succeeded=true`
- `youtube_video_id` present
- `youtube_url` present
- `visibility=private` or `visibility=unlisted`
- `external_api_called=true`
- `youtube_upload_executed=true`
- `uploaded=true`

If YouTube returns no video id, the result must be `succeeded=false`. Fake success is forbidden.

## YouTube Studio Check

After a successful smoke, verify manually in YouTube Studio:

- The video is private or unlisted as requested.
- The title is the smoke title.
- The description includes the affiliate disclosure and affiliate URL.
- The video did not become public automatically.

Do not change the video to public during this smoke.
